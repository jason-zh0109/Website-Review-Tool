# Database Model and Correlation

This model is based on an **AWS RDS instance using the db.t3.micro instance class with the MySQL engine**. The focus is on the `auth_user` table, which stores essential user authentication information, with other tables being auto-generated by Django during migrations. 

## auth_user Table

The `auth_user` table is a critical part of the user authentication system, primarily responsible for storing user credentials like usernames, emails, and passwords. Below is a breakdown of its key fields:

### Fields:

- **id**: An auto-incrementing primary key that uniquely identifies each user in the system.
- **username**: Stores the user's login name, restricted by a custom `RegexValidator` ensuring usernames are 5-20 characters long and only contain letters, numbers, underscores, or hyphens.
- **email**: Stores the user’s email address, validated to ensure proper format and uniqueness.
- **password**: Stores the hashed version of the user's password for security purposes.
- **is_active**: A boolean field that indicates whether the account is active.
- **is_superuser**: A boolean field that specifies if the user has admin access.
- **last_login**: Stores the timestamp of the user's last login.
- **date_joined**: The timestamp of when the user registered in the system.

## Correlations

Since the additional tables for managing permissions, user groups, and other relationships (such as `auth_user_groups`, `auth_user_user_permissions`, `auth_group`, etc.) are auto-created by Django during migration, they are not the main focus. These tables handle the backend logic for permissions and roles without requiring direct involvement in managing relationships or fields manually.

### Summary:

The main focus is on the `auth_user` table, which stores essential user credentials and status. While Django automatically generates the correlated tables during migrations, they provide necessary background functionality for user groups, permissions, and access control, which are managed automatically by Django's ORM.
